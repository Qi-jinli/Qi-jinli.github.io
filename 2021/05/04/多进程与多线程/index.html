多进程与多线程

发表于 2021-05-04 分类于 python

多任务

多任务

在同一时刻或同一时间段同时执行多个任务多任务

并发

任务数大于cpu核心数,cpu交替执行任务,并发（同一个时间段交替执行多个任务）

并行

同时时刻执行多个任务,并行(任务数小于cpu核心)

进程

进程是操作系统调度和分配资源的最小单位

程序开启之后默认开启一个进程

进程默认会有一个线程

多进程实现

  1. 导包
import multiprocessing
  1. 定义任务(函数)
def music():
    for i in range(3):
        print("music...")
        time.sleep(0.2)
  1. 创建进程(生产线)
coding_process = multiprocessing.Process(target=music) # target参数 需要跟上任务
  1. 开启进程
进程对象.start()

多任务面向对象实现

  1. 导包
import multiprocessing
  1. 继承multiprocessing.Process
class MyProcess(multiprocessing.Process):
    def __init__(self):
        # 调用父类的初始化方法
        super(MyProcess, self).__init__()
  1. 重写run方法,任务代码放到run方法中(run方法会自动调用,进程资源分配好自动调用,不需要自己调用)
# Process方法 进程的任务都是在run方法执行
    def run(self):
        for i in range(3):
            print("music...")
            time.sleep(0.2)
  1. 创建自定义进程对象
pro = MyProcess()
  1. 开启自定义进程
pro.start()

注意:

函数并没有进程和线程之分,在哪个进程调用,就运行在哪个进程中

进程传递参数

  1. 通过元组传递
multiprocessing.Process(target=coding, args=(3, "传智"))

注意:一个参数元组需要加, 顺序和参数顺序保持一致

  1. 通过字典方式传递
multiprocessing.Process(target=music, kwargs={"count": 2})

注意:元素键需要和函数形参名称保持一致

进程编号

获取进程编号

  1. 获取当前进程编号
os.getpid()
  1. 获取父进程进程编号
os.getppid()
  1. 获取当前进程名字
multiprocessing.current_process().name

进程间不共享数据

子进程创建把主进程进行拷贝,子进程使用的全局变量和主进程全局变量不相同

把进程由原来的并行执行变成串行执行

# join前代码
进程对象.join() 
# joint之后代码

注意：join之后的代码需要等到进程执行完之后再执行

主进程和子进程执行顺序

主进程会等到子进程执行结束之后再停止

如果想主进程代码执行结束结束子进程,可以通过守护进程或者手动停止进程

守护进程

进程对象.daemon = True
# 需要在进程start之前设置

手动停止子进程

进程对象.terminate()
# 在主进程代码执行结束时调用这个代码

进程命令

ps

ps -aux|grep 进程名

过滤正在运行感兴趣的进程

kill

kill -9 进程id

强制杀死进程

多线程创建

线程:CPU调度资源的最小单位

  1. 导入threading模块

    import threading

  2. 创建新线程,传递任务

    coding_thread = threading.Thread(target=coding)

  3. 开启线程

    coding_thread.start()

注意：start 发送指令,自动创建线程,run方法会运行在子线程

多线程传参

和多进程传递参数相同:args或者kwargs方式传递

args

以元组方式传递参数

coding_thread = threading.Thread(target=coding, args=(3,'传智'))

注意：函数多个参数,数据和函数形参顺序保持一致

kwargs

以字典的方式传递参数

music_thread = threading.Thread(target=music, kwargs={"count" : 1})

注意：字典的每个元素key是函数形参名称

线程执行顺序

守护线程

work_thread = threading.Thread(target=work, daemon=True)
work_thread.setDaemon(True)

线程一般不能直接杀死

停线程就是停止线程循环代码

可以设置终止变量

hasFinsh = False
# 工作函数
def work():
    for i in range(10):
        if hasFinsh:
            return
        print("work...")
        time.sleep(0.2)

线程间执行顺序

线程间和进程间执行都是无序

如果想让线程变成串行执行

sub_thread.join()

获取线程名称

threading.current_thread().name
